# I18n

Use this guide when adding or moving user-facing copy, validation messages, mailer text, controller notices, or view labels.

## File Layout

New translation files are organized by locale first, then feature:

```text
config/locales/<locale>/<feature>.yml
```

Examples:

```text
config/locales/fr/user.yml
config/locales/en/user.yml
config/locales/fr/event.yml
config/locales/en/event.yml
```

Do not create new files named with the locale suffix at the end, such as `user.fr.yml` or `event.en.yml`. Do not create feature folders such as `config/locales/models/` for new translations.

Legacy files can stay in place until they are intentionally refactored.

## Keys

The YAML root must match the locale folder:

```yml
fr:
  users:
    show:
      edit_profile: "Modifier mon profil"
```

Keep keys grouped by the feature that owns the copy. Prefer explicit scopes over broad generic keys.

## Model Errors

Model validation messages belong in the feature file for that model:

```text
config/locales/fr/user.yml
config/locales/en/user.yml
```

Use Rails' Active Record error scope:

```yml
fr:
  activerecord:
    errors:
      models:
        user:
          attributes:
            username:
              blank: "Le nom d'utilisateur est obligatoire."
```

## Rails Usage

Use Rails I18n helpers for user-facing copy:

```erb
<%= t(".edit_profile") %>
```

```ruby
redirect_to user_path(user), notice: t(".friend_request_sent", name: user.first_name)
```

Avoid hard-coded user-facing strings in views, controllers, mailers, and model validation messages when touching related code.
